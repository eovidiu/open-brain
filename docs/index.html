the problem

Your context is scattered across amnesiacs

Claude knows about Tuesday's architecture call. ChatGPT knows your writing style. Cursor knows the codebase quirks. None of them share, and each one forgets the moment the context window slides. So you re-explain — again.

Open Brain flips ownership: the memory belongs to you, not the assistant. Any tool that speaks the Model Context Protocol plugs into the same brain. The idea comes from Nate B Jones's essay "Every AI You Use Forgets You — Here's the Fix".

how it works

The life of a thought

From capture to recall in four steps — with an honest answer for what happens when things fail.

1 · Capture

A thought arrives at the capture Worker — from a webhook, a script, or an assistant's capture_memory tool. Two auth paths: a signed JWT, or an HMAC signature over timestamp.body with a 5-minute replay window.

POST → 201 · your text is never rejected, never mutated

2 · Understand

In parallel, the text becomes a 1536-dimension embedding (OpenAI) and gets structured metadata from an LLM — type, topics, people, action items, sentiment. Nine memory types, from decision to meeting_note.

embedding + metadata run concurrently — neither blocks storage

3 · Store

Everything lands in one Postgres table in your Neon project — raw text verbatim, vector indexed with HNSW, metadata as JSON. Plain SQL, no ORM, one database role. Your data has exactly one home.

memories · pgvector · single source of truth

4 · Recall — from anywhere

Claude Desktop talks to a local stdio server; everything else connects to the MCP Worker over Streamable HTTP with a bearer token. Same five tools either way: search by meaning, list, stats, capture, delete.

cosine similarity search · ranked · filterable by type & time
And when an AI call fails mid-capture?

Your thought is stored anyway, flagged pending or degraded. A janitor Worker wakes every minute, finds incomplete memories, and finishes the job — with exponential backoff and a hard stop at 10 attempts. Durability beats perfection: raw text is never lost.

under the hood

Slightly more technical

Three tiny Workers on Cloudflare's edge, one local process, one database. The whole thing runs on free tiers — the only metered cost is ~$0.40/month of AI calls at twenty thoughts a day.

Capture Worker

The HTTPS intake. Verifies JWT HS256 or replay-protected HMAC, rate-limits per credential, orchestrates the AI calls, writes the row. Returns 201 no matter which AI call flaked.

open-brain-capture

Retry Worker

A cron janitor, every minute. No public route at all — its HTTP handler answers 404 to everyone. Eligibility comes from one SQL function; terminal failure is flipped atomically in the database.

cron * * * * * · no HTTP

MCP Worker

Remote memory for any MCP client, over Streamable HTTP. /auth/token trades a client secret (constant-time compared, rate-limited) for a one-hour JWT. Stateless by design — a fresh MCP server per request.

open-brain-mcp · Agents SDK

Local stdio server

Claude Desktop launches it as a child process — no network, no auth theater, same five tools. It refuses to start if its embedding model disagrees with the database's, so the vector space can never silently fork.

mcp-server --stdio

One shared library

All three Workers import the same package for AI calls, validation, and queries. It exists because three hand-copied versions once drifted — and the drift caused a real incident. Now the divergence class is gone.

workers/shared · ~330 tests repo-wide

Security posture

Replay windows on webhooks, alg:none rejected, constant-time secret comparison, API keys redacted from every persisted error, CORS closed. Stored text is treated as data, never as instructions.

spec §9 · threat model included

run it yourself

Your database. Your keys. One wizard.

Free Neon + Cloudflare accounts, an OpenAI key, Node 18+. The 8-step wizard provisions everything — migrations, Workers, secrets, Claude Desktop — and is safe to re-run.

# clone & install
git clone https://github.com/eovidiu/open-brain.git && cd open-brain
npm ci

# one-time Cloudflare auth, then the wizard
npx wrangler login
npm run dev --workspace=cli

# later: health check across .env, Neon, and both Workers
npm run dev --workspace=cli status
